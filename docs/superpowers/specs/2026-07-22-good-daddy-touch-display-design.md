# Good Daddy 触摸显示屏设计

## 目标与范围

制作一个 7 英寸横向触摸显示屏，显示亲情点、当前日期、记录日期和称号。点击屏幕按钮后记录当天日期，并按已确认的亲情点规则更新称号。

本设计只冻结硬件选择、状态规则、界面和验收条件。固件编码、外壳建模和采购在硬件到手后开始。

## 已确认决策

- 显示屏：7 英寸、800×480、横向、电容触摸。
- 界面：A 均衡布局；主文字区域只显示称号。
- 供电：USB 长期供电。
- 时间：Wi-Fi 连接 NTP 获取当前日期；断网重启后等待校时，不使用不可信日期结算。
- 保存：断电不丢失亲情点和记录日期。
- 首次成功校时：`X=0`，记录日期显示“尚未记录”。

## 硬件方案

选择 Waveshare `ESP32-S3-Touch-LCD-7`，购买 **With Touch Function** 版本（厂商 SKU `27078`）。它把 ESP32-S3、7 英寸 IPS TFT、5 点电容触摸、Wi-Fi、16MB Flash 和 8MB PSRAM 集成在一块板上。

板卡触摸版本的最大平面尺寸为 `192.96 × 110.76 mm`。外壳以 `205 × 125 mm` 作为起始外轮廓，厚度按板卡与 USB-C 接头实际尺寸确定。

| 物料 | 规格 | 作用 |
|---|---|---|
| 主板 | ESP32-S3-Touch-LCD-7，带触摸版 | 显示、触摸、控制、Wi-Fi 与存储 |
| USB 电源 | 5V USB-C 电源及线缆 | 日常供电与烧录 |
| 外壳 | 3D 打印或激光切割 | 固定板卡，预留 USB-C 开孔 |

厂商页面显示该型号官方售价区间为 `$31.99–$36.99`（不含运费和税费）。本项目只使用 USB 供电；断电后的数据保存由 ESP32 NVS Flash 负责。

## 软件选型

采用 **Arduino-ESP32 + LVGL**：优先复用厂商提供的 Arduino、触摸和 LVGL 示例，再实现单页界面与状态保存。

未选择裸 TFT 屏加独立 ESP32-S3，因为 800×480 RGB 屏需要额外处理排线、背光供电、显示时序和触摸控制器，节省的物料成本不抵调试成本。未选择树莓派 HDMI 方案，因为它的成本、功耗和系统维护都高于本项目需求。

## 状态与持久化

只保存三个业务字段，均写入 ESP32 NVS Flash：

| 字段 | 类型 | 含义 |
|---|---|---|
| `score_x` | 有符号整数 | 当前亲情点 `X` |
| `last_settlement_date` | 本地日期 | 上一次已完成日结算的日期 |
| `record_date` | 可空本地日期 | 最后一次按下记录按钮的日期，仅显示，不参与计分 |

本地日期以 `Asia/Shanghai` 时区的 `YYYY-MM-DD` 表示。Wi-Fi 密码通过首次启动配网页面保存到设备，不写入源码或仓库。

### 日结算

设备在每次成功校时后比较当天与 `last_settlement_date`：

```text
days = 今天 - last_settlement_date
如果 days > 0：
  X = X + days
  last_settlement_date = 今天
  立即保存
```

因此设备跨日运行时会在凌晨结算；设备断电数天后重启，也会在 NTP 校时后一次补齐相隔天数。`record_date` 不参与分数计算。

### 记录按钮

按钮文字为“记录一次”。在时间已经成功校准时才可点击：

```text
record_date = 今天
如果 X > 0：
  X = 0
否则：
  X = X - 1
立即保存
```

因此，正亲情点时第一次记录归零；`X=0` 或负数时，每次点击都减 1。这个规则使亲情点连续变化，例如 `-2 → -1 → 0 → 1`。

触摸事件使用 LVGL 的单次点击事件，额外忽略 300ms 内的重复触发，避免一次触摸被记录多次。

### 无网络与断电行为

- 已运行的设备短时断网：继续使用已校准的系统时钟，正常显示与结算。
- 断网后重启：显示“等待校时”，禁用记录按钮；NTP 成功后再执行日结算。
- 任何按钮变更和日结算都立即写 NVS；因此 USB 拔出或重启不会重置已保存状态。

## 称号和颜色

称号依据 `X`，颜色同时作用于称号主区域与亲情点区域。颜色不作为唯一信息来源，文字称号始终显示。

| X 范围 | 称号 | 色彩 |
|---:|---|---|
| `X ≤ -6` | 恶魔在世 | 深红 `#7F1D1D` |
| `-5 ≤ X ≤ -3` | 糟糕爸爸 | 砖红 `#C2410C` |
| `-2 ≤ X ≤ -1` | 差劲爸爸 | 棕橙 `#B45309` |

## 界面：A 均衡布局

分辨率为 `800×480`，横屏。所有文字使用中文。

```text
┌──────────────────────────────────────────────────────┐
│ 当前日期：2026-07-22          记录日期：2026-05-11   │
├──────────────────────────────────────────────────────┤
│                                                      │
│                                                     │
│                     史 诗 爸 爸                     │
│                                                     │
│                                                      │
│                       亲情点：72                     │
│                                                      │
├──────────────────────────────────────────────────────┤
│                    [ 记录一次 ]                      │
└──────────────────────────────────────────────────────┘
```

称号是屏幕的主角：中间区域至少占屏幕高度的 55%，使用 128px 中文粗体字体，只显示当前称号。亲情点在称号下方以 40px 字体显示；日期固定为顶部两个区域，未有记录时右侧显示“记录日期：尚未记录”。按钮至少占屏幕宽度的 70%，高度不少于 56px，方便触摸。

## 验收条件

1. 首次校时后显示当前日期、`记录日期：尚未记录`、`亲情点：0` 与“普通爸爸”。
2. `X=10` 时点击一次，`X` 变为 `0`，记录日期变为当天，称号变为“普通爸爸”。
3. `X=0` 时连续点击两次，`X` 依次变为 `-1`、`-2`，称号均为“差劲爸爸”。
4. `X=-2` 时跨过一个自然日且不点击，`X` 变为 `-1`；再跨日，依次为 `0`、`1`。
5. 设备断电后重启，NTP 校时完成后恢复上次保存的 `X`、记录日期，并补齐漏掉的日结算。
6. 触摸按钮一次点击只产生一次状态变更。
7. 每个称号边界均显示正确文字和对应颜色。
8. 无网络重启时，不修改 `X` 和记录日期，且记录按钮不可用，直至成功校时。
